# Changelog

## 0.5.2 — 2026-07-19

- 将 API 音频转录总超时从 15 分钟延长到 1 小时（3600 秒），适合长视频和较慢的 ASR 服务。

## 0.5.1 — 2026-07-19

- 改进 ASR API 连接失败提示：针对端口未监听的 `WinError 10061` 显示 CrisperWeaver 启动条件和 `/health` 检查命令。

## 0.5.0 — 2026-07-19

- 新增 OpenAI 兼容 ASR API 后端，可连接 CrisperWeaver / MiMo 等本地服务。
- 新增 API 地址、密钥设置和 `/health` 连接测试；默认 `http://127.0.0.1:8765/v1`、`mimo-asr`、`zh`。
- 支持 `verbose_json` 分段响应和纯文本响应；API 请求取消、超时和错误会显示明确提示。
- 自动检测在本地 ASR 不可用时会尝试 API；手动选择 API 时不会启动本地 Python ASR。

## 0.4.0 — 2026-07-19

- 新增批量处理：从混合文本识别 B站链接、BV 号和 av 号，按出现顺序去重。
- 支持 1–4 路并行提取每个视频的全部分P，并分别导出独立 Markdown 文件。
- 单个视频失败不会中断其他任务；同名文件自动追加序号，支持取消批量任务。

## 0.3.0 — 2026-07-16

- 新增标准 Windows 安装包，支持开始菜单、可选桌面快捷方式和完整卸载。
- 新增“检测方式”，逐分P显示公开、匿名、登录浏览器和 ASR 是否可用。
- 新增五种精确模式：智能、只用公开、只用匿名、只用登录浏览器、只用本地 ASR。
- 所有来源最多尝试两次，第一次失败后固定等待 1 秒再重试。
- 移除底层 HTTP 隐式重试，确保单个来源不会在两次来源尝试之外额外请求。
- 重写智能排序：人工中文字幕 → B站中文 AI 字幕 → 其他 B站字幕 → 本地 ASR。
- 公开英文字幕不再阻止匿名或登录浏览器中的中文 AI 字幕。
- 字幕 URL 下载失败或正文为空时会重试并继续下一候选，不再提前终止整个分P。
- 新增真实登录状态显示及“检查登录”操作。

## 0.2.0 — 2026-07-16

- 移除 Codex 浏览器桥接和 `localhost:3456` 依赖。
- 新增独立 Edge / Chrome / Brave 专用登录浏览器。
- 新增标准库实现的本机 Chrome DevTools WebSocket 客户端。
- 固定字幕回退规则：公开视频接口 → 匿名播放器接口 → 专用登录浏览器 → 本地 ASR。
- 登录信息使用独立浏览器配置目录，不读取用户默认浏览器 Cookie。

## 0.1.0 — 2026-07-16

- 初始桌面界面、分P提取、本地 ASR 与四种导出格式。
